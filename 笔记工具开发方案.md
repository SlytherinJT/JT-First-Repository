# 简易笔记工具开发方案

## 一、项目概述

### 1.1 项目背景
开发一个轻量级的日常笔记记录工具，满足个人日常记录、整理、检索笔记的需求。本项目采用前后端分离架构，通过 Web 界面提供笔记的增删改查功能。

### 1.2 项目目标
- 实现笔记的创建、编辑、删除、分类检索等核心功能
- 提供简洁、响应式的用户界面
- 支持 Markdown 语法编辑
- 支持笔记标签分类和全文搜索

### 1.3 技术栈选择

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Vue 3 + Vite | 轻量、高效、组合式API |
| UI组件库 | Element Plus | 成熟、文档完善 |
| 后端框架 | Node.js + Express | JavaScript全栈、快速开发 |
| 数据库 | SQLite（better-sqlite3） | 轻量、无需单独部署 |
| 前端Markdown | marked + highlight.js | 渲染Markdown和代码高亮 |
| 包管理 | npm | 标准包管理工具 |

### 2.2 数据模型设计

```sql
-- 笔记表
CREATE TABLE notes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL DEFAULT '',
    content TEXT NOT NULL DEFAULT '',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 标签表
CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE
);

-- 笔记-标签关联表
CREATE TABLE note_tags (
    note_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    PRIMARY KEY (note_id, tag_id),
    FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);
```

### 2.3 API接口设计

| 方法 | 路径 | 功能 | 请求体/参数 |
|------|------|------|-------------|
| GET | /api/notes | 获取笔记列表 | ?search=&tag=&page=&limit= |
| GET | /api/notes/:id | 获取单条笔记 | — |
| POST | /api/notes | 创建笔记 | {title, content, tags[]} |
| PUT | /api/notes/:id | 更新笔记 | {title, content, tags[]} |
| DELETE | /api/notes/:id | 删除笔记 | — |
| GET | /api/tags | 获取所有标签 | — |
| POST | /api/tags | 创建标签 | {name} |
| DELETE | /api/tags/:id | 删除标签 | — |

### 6.1 分支管理策略

```
main        主分支（稳定版本）
  └── dev   开发分支（日常开发）
       └── feature/ui-optimize   （功能分支示例）
       └── fix/search-bug         （修复分支示例）
```

### 6.2 分支操作示例

```bash
# 1. 创建并切换到开发分支
git checkout -b dev

# 2. 在dev分支上开发新功能或修复
# 开发完成后提交
git add .
git commit -m "feat: 添加笔记排序功能"

# 3. 创建功能分支（处理更复杂的功能）
git checkout -b feature/search-optimization
# 开发完成后合并回dev
git checkout dev
git merge feature/search-optimization

# 4. dev分支稳定后合并到main
git checkout main
git merge dev
```

### 6.3 提交规范建议

| 类型 | 使用场景 | 示例 |
|------|---------|------|
| feat | 新功能 | `feat: 添加笔记标签筛选功能` |
| fix | 修复bug | `fix: 修复笔记删除后列表未刷新问题` |
| docs | 文档更新 | `docs: 更新README使用说明` |
| style | 代码格式 | `style: 统一组件命名规范` |
| refactor | 代码重构 | `refactor: 拆分NoteList组件` |

### 7.3 后端静态文件服务配置

修改 `server/src/index.js`，添加静态文件服务：

```javascript
const express = require('express');
const path = require('path');
const app = express();

// 提供前端静态文件
app.use(express.static(path.join(__dirname, '../public')));

// 所有非API路由返回前端入口文件
app.get('*', (req, res) => {
  if (!req.path.startsWith('/api')) {
    res.sendFile(path.join(__dirname, '../public/index.html'));
  }
});
```
